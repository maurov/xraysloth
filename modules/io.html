get_axis_data(ax_name=None, to_energy=None)[source]¶

Get data for the scan axis

This method returns the data (=label and array) for a given axis of the selected scan. It is primarily targeted to a “scanning” axis, but any counter can be used. It is possible to control common conversions, like Bragg angle to energy.

Parameters

  • ax_name (str or None) –

  • to_energy (dict) –

    Controls the conversion of the signal to energy [None]

    Note

    Bragg angle assumed in mrad, output in eV

    {

    “bragg_ax”: “str”, #: name of counter used for Bragg angale “bragg_ax_type”: “str”, #: ‘motor’ or ‘counter’ “bragg_enc_units”: float, #: units to convert encoder to deg (bragg_ax should contain ‘enc’)

    }

Returns

Return type

label, data

get_signal_data(sig_name, mon=None, deglitch=None, norm=None)[source]¶

Get data for the signal counter

This method returns the data (=label and array) for a given signal of the selected scan. It is possible to control normalization and/or deglitching.

Order followed in the basic processing:

  • raw data

  • divide by monitor signal (+ multiply back by average)

  • deglitch

  • norm

Parameters

  • sig_name (str) –

  • mon (dict) –

    Controls the normalization of the signal by a monitor signal [None] {

    ”monitor”: “str”, #: name of counter used for normalization “cps”: bool, #: multiply back to np.average(monitor)

    }

  • deglitch (dict) – Controls sloth.math.deglitch.remove_spikes_medfilt1d() [None]

  • norm (dict) – Controls the normalization by given method

Returns

Return type

label, data

Export a selected list of scans to HDF5 file

Note

This is a simple wrapper to silx.io.convert.write_to_h5()

Parameters

  • scans (str, list of ints or list of lists (str/ints)) – scan numbers to export (parsed by sloth.utils.strings.str2rng) if a list of lists, scans_groups is required

  • fname_out (str) – output file name

  • scans_groups (list of strings) – groups of scans

  • h5path (str (optional)) – path inside HDF5 [None -> ‘/’]

  • overwrite (boolean (optional)) – force overwrite if the file exists [False]

  • conf_dict (None or dict (optional)) – configuration dictionary saved as ‘{hdfpath}/.config’
